# PL/SQL con Oracle
## Clase 1 — Introducción y Estructura de Bloques

Programación de Bases de Datos

---

# ¿Qué es PL/SQL?

**PL/SQL** = *Procedural Language / SQL*

Combina dos mundos complementarios:

| Lenguaje | Generación | Características |
|---|---|---|
| **SQL** | 4GL | Describe **qué** obtener; no **cómo** |
| **PL/SQL** | 3GL | Añade estructuras **procedimentales** |

## PL/SQL añade sobre SQL:
- Variables y tipos de datos propios
- Estructuras de control (IF, bucles)
- Procedimientos, funciones y paquetes
- Manejo de excepciones (errores)
- Tipos de objetos y métodos (desde versión 8)

---

# ¿Por qué PL/SQL y no solo SQL?

```sql
-- SQL: una consulta, un resultado
SELECT AVG(Salario) FROM Empleados WHERE Dpto = 10;
```

Con SQL puro **no podemos**:
- Tomar decisiones basadas en el resultado
- Repetir operaciones en bucle sobre filas
- Reutilizar lógica compleja como subprogramas
- Controlar errores de forma estructurada

```sql
-- PL/SQL: lógica procedimental completa
DECLARE
  Media NUMBER;
BEGIN
  SELECT AVG(Salario) INTO Media FROM Empleados WHERE Dpto = 10;
  IF Media < 1500 THEN
    UPDATE Empleados SET Salario = Salario * 1.1 WHERE Dpto = 10;
  END IF;
END;
```

---

# Ventajas de PL/SQL

## Integración con Oracle
- Se ejecuta **dentro del servidor** → muy eficiente
- Compilado una vez, ejecutado muchas veces (subprogramas)

## Minimiza tráfico cliente-servidor
- En lugar de enviar N consultas SQL, se envía **un único bloque**
- Especialmente relevante en redes lentas o con mucha carga

## Robustez
- Gestión de errores con **excepciones** (heredado de Ada)
- `%TYPE` y `%ROWTYPE` para adaptar variables a la BD automáticamente

<div class="callout">
💡 PL/SQL se basa en el lenguaje <strong>Ada</strong> (3GL). Muchos conceptos —bloques, excepciones, fuerte tipado— proceden de ahí.
</div>

# Elementos léxicos

## Caracteres y sensibilidad a mayúsculas
- PL/SQL utiliza el conjunto **ASCII**
- **No distingue** mayúsculas/minúsculas… salvo dentro de cadenas entre comillas simples

```sql
nombre := 'Oracle';   -- 'Oracle' ≠ 'ORACLE' (cadena literal)
SELECT * FROM pieza;  -- Equivalente a SELECT * FROM PIEZA
```

## Identificadores
- Empiezan con **letra**, seguida de letras, dígitos, `$`, `_`, `#`
- Longitud máxima: **30 caracteres**
- Entre **comillas dobles** permiten caracteres especiales o palabras reservadas:

```sql
"EXCEPTION"   -- Tabla que se llama así (palabra reservada)
"Mi Variable" -- Identificador con espacio (desaconsejado)
```

# Operadores y delimitadores clave

| Símbolo | Significado |
|---|---|
| `:=` | Asignación |
| `=` | Comparación de igualdad |
| `<>` o `!=` | Distinto de |
| `**` | Potenciación |
| `\|\|` | Concatenación de cadenas |
| `..` | Rango (ej. `1..10` en bucle FOR) |
| `%` | Indicador de atributo (`%TYPE`, `%ROWTYPE`, `%FOUND`…) |
| `;` | Terminador de orden |
| `--` | Comentario de una línea |
| `/* … */` | Comentario multilínea |

<div class="callout">
⚠️ El operador de <strong>asignación</strong> es <code>:=</code>, no <code>=</code>. El <code>=</code> solo se usa en comparaciones. Confundirlos es el error más frecuente al empezar.
</div>

# La estructura de un bloque PL/SQL

---

# Anatomía de un bloque PL/SQL

```sql
DECLARE
  -- Sección DECLARATIVA (opcional)
  -- Variables, tipos, cursores, subprogramas locales
  -- Visibles desde aquí hasta END

BEGIN
  -- Sección EJECUTABLE (obligatoria)
  -- Órdenes SQL y procedimentales
  -- Puede contener otros bloques anidados

EXCEPTION
  -- Sección de EXCEPCIONES (opcional)
  -- Gestión de errores

END;
/
```

<div class="callout">
La única sección <strong>obligatoria</strong> es <code>BEGIN … END</code>. Las otras dos son opcionales. El <code>/</code> final le indica a SQL*Plus que ejecute el bloque.
</div>

---

# Tipos de bloques

| Tipo | Descripción | Se almacena |
|---|---|---|
| **Anónimo** | Sin nombre, se ejecuta una vez | ❌ No |
| **Nominado** | Anónimo con etiqueta `<<nombre>>` | ❌ No |
| **Procedimiento** | `CREATE PROCEDURE` | ✅ Sí |
| **Función** | `CREATE FUNCTION`, devuelve valor | ✅ Sí |
| **Paquete** | Agrupa procedimientos y funciones | ✅ Sí |
| **Trigger** | Se dispara con eventos DML | ✅ Sí |

---

# Bloque anónimo — primer ejemplo

**Objetivo:** insertar dos piezas de tipo "Tornillo" con pesos distintos.

```sql
DECLARE
  NumP NUMBER := 7;   -- Número de pieza inicial
BEGIN
  INSERT INTO Pieza (P#, Nombre, Peso)
  VALUES (NumP, 'Tornillo', 2);

  INSERT INTO Pieza (P#, Nombre, Peso)
  VALUES (NumP + 1, 'Tornillo', 4);

  COMMIT;
END;
/
```

## Observaciones:
- `NumP` se declara e inicializa en DECLARE
- Las variables se usan directamente en las órdenes SQL
- El bloque confirma la transacción con `COMMIT`

---

# Bloque nominado

Añadimos una **etiqueta** para poder referirnos a él:

```sql
<<Bloque_Insertar>>
DECLARE
  NumP NUMBER := 7;
BEGIN
  INSERT INTO Pieza (P#, Nombre, Peso)
  VALUES (NumP, 'Tornillo', 2);

  INSERT INTO Pieza (P#, Nombre, Peso)
  VALUES (NumP + 1, 'Tornillo', 4);

END Bloque_Insertar;  -- Nombre opcional tras END, pero recomendable
/
```

## Utilidad de los nombres:
- En bloques **anidados**, si dos variables tienen el mismo nombre, se distinguen: `Bloque_Insertar.NumP`
- Mejora la **legibilidad** del código

# SQL*Plus: comandos esenciales

Antes de ejecutar código PL/SQL en SQL*Plus hay que configurar el entorno:

```sql
-- Activar la salida por pantalla (imprescindible para ver resultados)
SET SERVEROUTPUT ON SIZE 1000000

-- Cambiar el ancho de línea
SET LINESIZE 120

-- Ver errores de compilación del último objeto creado
SHOW ERR

-- Ejecutar una sentencia PL/SQL sin escribir un bloque completo
EXEC NombreProcedimiento(arg1, arg2)
```
---

# Un procedimiento almacenado

Los procedimientos se crean con `CREATE OR REPLACE` y se **almacenan en la BD**:

```sql
CREATE OR REPLACE PROCEDURE MostrarPieza7 AS
  NombrePieza VARCHAR2(50);
BEGIN
  SELECT Nombre INTO NombrePieza
  FROM Pieza
  WHERE P# = 7;

  DBMS_OUTPUT.PUT_LINE('Nombre de la pieza 7: ' || NombrePieza);
END MostrarPieza7;
/
```

Para ejecutarlo:

```sql
EXEC MostrarPieza7;
-- o desde otro bloque:
BEGIN
  MostrarPieza7;
END;
/
```

# Un disparador (trigger) — anticipo

Los triggers se disparan **automáticamente** ante eventos DML:

```sql
CREATE OR REPLACE TRIGGER PesoPositivo
  BEFORE INSERT OR UPDATE OF Peso ON Pieza
  FOR EACH ROW
BEGIN
  IF :new.Peso < 0 THEN
    RAISE_APPLICATION_ERROR(-20100, 'El peso no puede ser negativo');
  END IF;
END PesoPositivo;
/
```

## Elementos que vemos:
- `BEFORE INSERT OR UPDATE` → cuándo se activa
- `FOR EACH ROW` → se ejecuta una vez por fila afectada
- `:new.Peso` → valor que se va a insertar/actualizar
- Los estudiaremos en profundidad en la **Clase 7**

# Comentarios en PL/SQL

## Una línea
```sql
-- Esto es un comentario de una línea
NumP := 7; -- Comentario al final de una línea de código
```

## Multilínea (estilo C)
```sql
/*
  Este bloque calcula el salario medio
  de los empleados del departamento 10.
  Autor: ..., Fecha: ...
*/
BEGIN
  ...
END;
```

<div class="callout">
Comenta siempre: qué hace cada bloque, qué restricciones asumes, qué significa cada argumento. Los comentarios son parte del código profesional.
</div>
