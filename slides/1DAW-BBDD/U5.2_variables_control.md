# PL/SQL con Oracle
## Clase 2 — Variables, Tipos de Datos y Estructuras de Control

Programación de Bases de Datos

---

# Variables en PL/SQL — Declaración

Todas las variables se declaran en la sección `DECLARE`:

```sql
nombre  tipo  [[CONSTANT | NOT NULL] := valor_inicial];
```

## Ejemplos

```sql
DECLARE
  Contador    NUMBER          := 0;         -- Inicializada a 0
  Nombre      VARCHAR2(100);                -- Valor por defecto: NULL
  Pi          CONSTANT NUMBER := 3.14159;   -- No puede cambiar
  Edad        NUMBER NOT NULL := 18;        -- NULL no permitido
  FechaHoy    DATE            := SYSDATE;   -- Fecha actual
  EsActivo    BOOLEAN         := TRUE;
BEGIN
  Contador := Contador + 1;
  DBMS_OUTPUT.PUT_LINE('Contador: ' || Contador);
END;
/
```

<div class="callout">
Si no se especifica valor inicial, la variable vale <code>NULL</code> (excepto con <code>NOT NULL</code>, que obliga a inicializar).
</div>

---

# Tipos escalares — Numéricos

## `NUMBER(precisión, escala)` — Numérico real
```sql
Precio      NUMBER(10, 2);   -- Hasta 10 dígitos, 2 decimales
Total       NUMBER;           -- Precisión máxima Oracle (38 dígitos)
```
- Se almacena en formato **decimal** → conversión a binario al operar
- Subtipos compatibles: `INTEGER`, `INT`, `DECIMAL`, `REAL`, etc.

## `BINARY_INTEGER` — Entero binario
```sql
Contador    BINARY_INTEGER := 0;   -- Complemento a 2, rango ±2147483647
```
- **Más rápido** para operaciones aritméticas (sin conversión decimal)
- Ideal para **contadores** e índices
- Subtipos: `NATURAL` (≥0), `POSITIVE` (≥1), `SIGNTYPE` (-1, 0, 1)

<div class="callout">
Usa <code>BINARY_INTEGER</code> para contadores e índices; <code>NUMBER</code> para datos monetarios o con decimales.
</div>

---

# Tipos escalares — Texto, Fecha y Lógico

## Cadenas de caracteres
```sql
Nombre      VARCHAR2(100);  -- Longitud variable, máx 32.767 bytes en PL/SQL
Codigo      CHAR(5);        -- Longitud fija, se rellena con blancos
```
<div class="warning">
⚠️ En la BD, <code>VARCHAR2</code> admite hasta 4.000 bytes. En PL/SQL hasta 32.767. Si asignas una variable PL/SQL a una columna de BD y supera 4.000, obtendrás un error.
</div>

## Fecha
```sql
Hoy     DATE := SYSDATE;   -- Almacena: siglo, año, mes, día, hora, min, seg
```

## Lógico (solo en PL/SQL, no en SQL)
```sql
EsValido    BOOLEAN := TRUE;  -- Valores: TRUE, FALSE, NULL
```
- `BOOLEAN` **no existe como tipo de columna** en Oracle SQL
- Solo puede usarse en variables y argumentos de PL/SQL

---

# `%TYPE` — Herencia de tipo

En lugar de especificar el tipo manualmente, se **hereda de la BD**:

```sql
DECLARE
  -- Tipo de la variable = tipo de la columna P# en la tabla Pieza
  NumPieza    Pieza.P#%TYPE;

  -- Tipo heredado de otra variable ya declarada
  OtraPieza   NumPieza%TYPE;

  -- Con tablas de la BD, siempre actualizado automáticamente
  NombreEmp   Empleados.Nombre%TYPE;
BEGIN
  SELECT P# INTO NumPieza FROM Pieza WHERE Nombre = 'Tornillo';
  DBMS_OUTPUT.PUT_LINE('Pieza: ' || NumPieza);
END;
/
```

## ¿Por qué es importante?
- Si cambia el tipo de la columna en la BD, el código **sigue funcionando** sin modificarlo
- Evita errores de truncamiento o incompatibilidad
- La restricción `NOT NULL` **no** se hereda

# Lógica trivaluada y estructura IF

---

# Lógica trivaluada: TRUE, FALSE, NULL

En PL/SQL (como en SQL), `NULL` significa **"desconocido"**, no cero ni vacío.

## Tablas de verdad

| `A` | `NOT A` |   | `A AND B` | T | N | F |   | `A OR B` | T | N | F |
|-----|---------|---|-----------|---|---|---|---|----------|---|---|---|
| T | F |   | **T** | T | N | F |   | **T** | T | T | T |
| N | N |   | **N** | N | N | F |   | **N** | T | N | N |
| F | T |   | **F** | F | F | F |   | **F** | T | N | F |

<div class="warning">
⚠️ Cualquier comparación con NULL devuelve NULL (no TRUE ni FALSE). Por eso <code>IF (A = NULL)</code> nunca es TRUE. Usa siempre <code>IS NULL</code> o <code>IS NOT NULL</code>.
</div>

---

# Estructura IF — ELSIF — ELSE

```sql
IF condicion1 THEN
  secuencia_de_ordenes_1;
ELSIF condicion2 THEN
  secuencia_de_ordenes_2;
ELSIF condicion3 THEN
  secuencia_de_ordenes_3;
ELSE
  secuencia_de_ordenes_n;
END IF;
```

- `ELSIF` y `ELSE` son **opcionales**; puede haber tantos `ELSIF` como se necesite
- Si la condición es `FALSE` **o** `NULL`, no se ejecuta esa rama → pasa a la siguiente

<div class="callout">
Nótese que es <code>ELSIF</code> (sin segunda E), no <code>ELSEIF</code> como en otros lenguajes.
</div>

---

# IF con NULL — error clásico

```sql
-- ⚠️ PELIGROSO si A o B pueden ser NULL
IF A < B THEN
  C := 1;
ELSE
  C := 2;       -- Se ejecuta también cuando A o B son NULL
END IF;
```

```sql
-- ❌ NO equivalente: si A>=B es NULL, C queda sin asignar
IF A >= B THEN
  C := 2;
ELSE
  C := 1;
END IF;
```
---

```sql
-- ✅ CORRECTO: comprobamos primero si hay nulos
IF A IS NULL OR B IS NULL THEN
  C := NULL;    -- O el valor que corresponda
ELSIF A < B THEN
  C := 1;
ELSE
  C := 2;
END IF;
```

---

# Ejemplo IF completo

**Clasificar un hotel según su número de habitaciones:**

```sql
DECLARE
  NumHabitaciones Hotel.Nhabs%TYPE;
  Categoria       VARCHAR2(40);
BEGIN
  SELECT Nhabs INTO NumHabitaciones
  FROM Hotel WHERE HotelID = 99;

  IF NumHabitaciones IS NULL THEN
    Categoria := 'Desconocido';
  ELSIF NumHabitaciones < 50 THEN
    Categoria := 'Pequeño';
  ELSIF NumHabitaciones < 100 THEN
    Categoria := 'Mediano';
  ELSE
    Categoria := 'Grande';
  END IF;

  DBMS_OUTPUT.PUT_LINE('Hotel 99: ' || Categoria);
END;
/
```

# Bucles: LOOP, WHILE y FOR

---

# Bucle LOOP (infinito con EXIT)

```sql
LOOP
  secuencia_de_ordenes;
  EXIT WHEN condicion;   -- Sale cuando condicion es TRUE
END LOOP;
```

Son equivalentes:
```sql
EXIT WHEN Cont > 10;
-- es lo mismo que:
IF Cont > 10 THEN EXIT; END IF;
```
---

**Ejemplo:** insertar valores del 1 al 10

```sql
DECLARE
  Cont BINARY_INTEGER := 1;
BEGIN
  LOOP
    INSERT INTO Tabla_Temp VALUES (Cont);
    Cont := Cont + 1;
    EXIT WHEN Cont > 10;
  END LOOP;
END;
/
```

---

# Bucle WHILE

```sql
WHILE condicion LOOP
  secuencia_de_ordenes;
END LOOP;
```

- Se ejecuta **cero o más veces**
- La condición se evalúa **antes** de cada iteración
- Si la condición es `FALSE` o `NULL` desde el principio, no entra

**Mismo ejemplo con WHILE:**

```sql
DECLARE
  Cont BINARY_INTEGER := 1;
BEGIN
  WHILE Cont <= 10 LOOP
    INSERT INTO Tabla_Temp VALUES (Cont);
    Cont := Cont + 1;
  END LOOP;
END;
/
```

---

# Bucle FOR

```sql
FOR contador IN [REVERSE] limite_inf .. limite_sup LOOP
  secuencia_de_ordenes;
END LOOP;
```

**Características importantes:**
- El contador se declara **automáticamente** como `BINARY_INTEGER`; no se declara en DECLARE
- El contador **no puede modificarse** dentro del bucle
- Con `REVERSE` recorre de `Sup` a `Inf`
- Los límites pueden ser constantes, variables o expresiones

**Mismo ejemplo con FOR — versión más limpia:**

```sql
BEGIN
  FOR Cont IN 1 .. 10 LOOP
    INSERT INTO Tabla_Temp VALUES (Cont);
  END LOOP;
END;
/
```

---

# Comparación de los tres bucles

| | LOOP | WHILE | FOR |
|---|---|---|---|
| ¿Cuándo evalúa? | Al final (mínimo 1 vez) | Al principio (puede ser 0 veces) | Automático |
| ¿Hay contador explícito? | Sí, declarado | Sí, declarado | No, automático |
| ¿Cuándo usar? | Cuando no sabes cuántas veces | Cuando dependes de una condición | Cuando conoces el rango |

<div class="callout">
Para recorrer un rango numérico conocido, usa siempre el bucle <strong>FOR</strong>: es más conciso y no permite errores de actualización del contador.
</div>

---

# Bucles anidados con etiquetas

```sql
<<BucleExterno>>
FOR i IN 1 .. 3 LOOP
  <<BucleInterno>>
  FOR j IN 1 .. 3 LOOP
    IF i = j THEN
      -- Podemos salir del bucle externo desde dentro
      EXIT BucleExterno WHEN i = 2;
    END IF;
    DBMS_OUTPUT.PUT_LINE('i=' || i || ', j=' || j);
  END LOOP BucleInterno;
END LOOP BucleExterno;
```

## Reglas:
- `EXIT` sin etiqueta sale del bucle **más interno**
- `EXIT nombre` sale del bucle con esa **etiqueta**
- Las etiquetas también ayudan a distinguir variables con el mismo nombre: `BucleExterno.i`

# Resumen de la clase

| Concepto | Clave |
|---|---|
| `NUMBER` | Numérico decimal, general |
| `BINARY_INTEGER` | Entero binario, rápido, para contadores |
| `VARCHAR2(n)` | Cadena variable, hasta 32.767 en PL/SQL |
| `BOOLEAN` | Solo en PL/SQL (no en columnas de BD) |
| `%TYPE` | Hereda el tipo de una columna o variable |
| `NULL` | "Desconocido" — lógica trivaluada |
| `IF/ELSIF/ELSE` | Comprueba nulos antes de comparar |
| `LOOP` | Sale con `EXIT WHEN`; siempre ejecuta al menos una vez |
| `WHILE` | Condición al inicio; puede no ejecutarse |
| `FOR` | Rango conocido; contador automático |
