# 1. Conceptos Fundamentales

## ¿Qué es SQL?

**SQL** (Structured Query Language) es el lenguaje estándar para acceder y manipular datos en sistemas gestores de bases de datos relacionales (SGBD).

**Características principales:**
- **No procedural**: indicas *qué* quieres obtener, no *cómo* obtenerlo
- **Actúa sobre conjuntos de filas**, no sobre registros individuales
- **Base teórica**: se fundamenta en el **álgebra relacional** y el **cálculo relacional**
- Trabaja con **tablas, columnas y filas** (no con relaciones, atributos y tuplas puras)

> **Nota importante**: Una tabla SQL **puede tener filas repetidas**, a diferencia de una relación matemática pura.

---

## Tipos de Sentencias SQL

| Categoría | Descripción | Ejemplos |
|-----------|-------------|----------|
| **DML** (Lenguaje de Manipulación de Datos) | Consultar, insertar, modificar y eliminar datos | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
| **DDL** (Lenguaje de Definición de Datos) | Crear, modificar y eliminar estructuras | `CREATE TABLE`, `ALTER TABLE`, `DROP TABLE` |
| **DCL** (Lenguaje de Control de Datos) | Control de transacciones, seguridad y concurrencia | `COMMIT`, `ROLLBACK`, `GRANT`, `REVOKE` |

> Esta presentación se centra en **DML** y consultas avanzadas.

# 2. Expresiones y Predicados

## Definición de Expresión

Una **expresión** es una formulación de una secuencia de operaciones que, al ejecutarse, **devuelve un único valor**.

**Elementos que pueden formar una expresión:**
- Constantes (valores fijos)
- Nombres de columna
- Funciones
- Otras expresiones
- Paréntesis para modificar precedencia

**Operadores permitidos:**
| Tipo | Operadores |
|------|------------|
| Numéricos | `+`, `-`, `*`, `/` |
| Alfanumérico | `||` (concatenación) |

---

```sql
-- Ejemplos de expresiones válidas
3 + 2
'A' || 'BC'               -- Resultado: 'ABC'
SAL * 1.5                 -- Salario incrementado 50%
0.5 * COMM                -- La mitad de la comisión
SAL + COMM                -- Salario total
```

> **Regla**: Si un operando de una expresión es **NULL**, el resultado de toda la expresión es **NULL**.

---

## Operaciones con NULL

El valor **NULL** representa la **ausencia de información**. Debe diferenciarse claramente de:
- El valor **0** (cero numérico conocido)
- La cadena vacía **''** (cadena de longitud cero)

**Consecuencias fundamentales:**

| Operación | Resultado |
|-----------|-----------|
| `NULL + 5` | `NULL` |
| `NULL * 10` | `NULL` |
| `'Hola' || NULL` | `NULL` |
| `NULL = NULL` | `NULL` (¡no TRUE!) |
| `NULL != NULL` | `NULL` (¡no TRUE!) |

> **Regla de oro con NULL**: Cualquier operación aritmética o de comparación con NULL produce **NULL**, no TRUE ni FALSE.

---

## Definición de Predicado

Un **predicado** es una condición que se evalúa sobre cada fila y cuyo resultado puede ser:
- **TRUE** (verdadero)
- **FALSE** (falso)
- **NULL** (desconocido)

> SQL utiliza una **lógica de tres valores** (3VL - Three-Valued Logic), a diferencia de la lógica booleana clásica de dos valores.

**Uso de predicados:**
- En la cláusula `WHERE`: filtra filas individuales
- En la cláusula `HAVING`: filtra grupos de filas

---

## Predicados Simples

### 1. De comparación
Comparan dos expresiones con operadores: `<`, `>`, `=`, `!=` (o `<>`), `<=`, `>=`

```sql
SELECT * FROM emp WHERE sal <= 1900;
```

### 2. De valor nulo
> **¡IMPORTANTE!** Los predicados de comparación (`=`, `!=`) **NO sirven** para detectar NULL.

```sql
-- INCORRECTO: nunca devolverá TRUE
SELECT * FROM emp WHERE comm = NULL;

-- CORRECTO
SELECT * FROM emp WHERE comm IS NULL;
SELECT * FROM emp WHERE comm IS NOT NULL;
```

---

## Predicados Simples (continuación)

### 3. De rango: BETWEEN
Comprueba si un valor está dentro de un rango **inclusive**:

```sql
SELECT * FROM emp WHERE sal BETWEEN 1500 AND 3000;
-- Equivalente a: sal >= 1500 AND sal <= 3000
```

### 4. De pertenencia: IN
Comprueba si un valor coincide con alguno de una lista:

```sql
SELECT * FROM emp WHERE deptno IN (10, 30);
-- Equivalente a: deptno = 10 OR deptno = 30
```

---

### 5. De patrón: LIKE
Usa comodines para búsquedas de texto:

| Comodín | Significado |
|---------|-------------|
| `_` | Un único carácter |
| `%` | Cualquier cadena (incluida vacía) |

```sql
SELECT * FROM emp WHERE ename LIKE '%NE%';
SELECT * FROM emp WHERE ename LIKE 'S___';  -- 4 letras, empieza por S
```

---

## Predicados Compuestos

Combinan predicados simples con operadores lógicos: **AND**, **OR**, **NOT**

**Tabla de verdad completa (lógica de tres valores):**

| X | Y | X AND Y | X OR Y | NOT X |
|---|---|---------|--------|-------|
| TRUE | TRUE | TRUE | TRUE | FALSE |
| TRUE | FALSE | FALSE | TRUE | FALSE |
| TRUE | NULL | NULL | TRUE | FALSE |
| FALSE | TRUE | FALSE | TRUE | TRUE |
| FALSE | FALSE | FALSE | FALSE | TRUE |
| FALSE | NULL | FALSE | NULL | TRUE |
| NULL | TRUE | NULL | TRUE | NULL |
| NULL | FALSE | FALSE | NULL | NULL |
| NULL | NULL | NULL | NULL | NULL |

> **Consecuencia práctica**: Una condición con NULL no es ni verdadera ni falsa, por lo que la fila **no se incluye** en el resultado.

---

## Ejemplos de Predicados Compuestos

```sql
-- Empleados que ingresaron después de junio de 1981
-- O que tienen salario inferior a 1500
SELECT ename, sal, hiredate
FROM emp
WHERE hiredate > '1-jun-1981' OR sal < 1500
ORDER BY 3, 1;
```

```sql
-- Empleados que ganan más de 2500 en total (salario + comisión)
-- ¡OJO! Si comm es NULL, sal + comm será NULL
SELECT ename
FROM emp
WHERE sal > 2500 OR sal + comm > 2500;
```

> **Consejo**: Usa paréntesis para clarificar la precedencia cuando combines AND y OR.

# 3. La Sentencia SELECT

## Estructura Completa

```sql
SELECT [DISTINCT | ALL] {* | <expr1> [, <expr2>] ...}
FROM <tabla1> [, <tabla2>, ...]
[WHERE <condicion_where>]
[GROUP BY <group_expr1> [, <group_expr2>, ...]
 [HAVING <condicion_having>]]
[ORDER BY <expr_orderby1> [ASC | DESC] [, ...]];
```

---

## Orden de Ejecución de las Cláusulas

El orden de escritura NO es el orden de ejecución. El SGBD procesa las cláusulas en este orden:

| Orden | Cláusula | Función | Qué procesa |
|-------|----------|---------|-------------|
| 1 | `FROM` | Obligatoria | Determina la tabla o tablas de origen |
| 2 | `WHERE` | Opcional | Filtra **filas individuales** según condición |
| 3 | `GROUP BY` | Opcional | Agrupa filas con valores iguales en columnas de agrupamiento |
| 4 | `HAVING` | Opcional | Filtra **grupos** según condición |
| 5 | `SELECT` | Obligatoria | Selecciona y calcula las columnas a mostrar |
| 6 | `ORDER BY` | Opcional | Ordena el resultado final |

> **Clave**: `WHERE` filtra antes de agrupar; `HAVING` filtra después de agrupar.

---

## Ejemplos de SELECT

### Ejemplo 1: Selección completa
```sql
SELECT * FROM emp;
```
> `*` indica **todas las columnas** de la tabla.

### Ejemplo 2: Filtrado básico
```sql
SELECT empno, ename
FROM emp
WHERE deptno = 10;
```

| EMPNO | ENAME |
|-------|-------|
| 7782 | CLARK |
| 7839 | KING |

---

### Ejemplo 3: Ordenación
```sql
SELECT * FROM dept ORDER BY dname;
```
> `ORDER BY` ordena ascendentemente (ASC) por defecto. Usa `DESC` para descendente.

### Ejemplo 4: Orden jerarquizado
```sql
SELECT comm AS "COMISION", deptno AS DEPARTAMENTO, ename NOMBRE
FROM emp
WHERE sal < 1900
ORDER BY deptno, comm DESC;
```
> Primero ordena por departamento (ascendente), luego por comisión (descendente) dentro de cada uno.

---

### Ejemplo 5: DISTINCT
```sql
-- Sin DISTINCT: muestra combinaciones repetidas
SELECT job, TO_CHAR(hiredate, 'yyyy') CONTRATADO
FROM emp WHERE deptno = 30;

-- Con DISTINCT: elimina filas duplicadas
SELECT DISTINCT job, TO_CHAR(hiredate, 'yyyy') CONTRATADO
FROM emp WHERE deptno = 30;
```

> `DISTINCT` afecta a **toda la fila**, no a una sola columna. Dos valores NULL se consideran iguales con DISTINCT.

# 4. Funciones

## Funciones Escalares vs. de Agrupamiento

| Tipo | Descripción | Ejemplo |
|------|-------------|---------|
| **Escalares** | Devuelven un valor **por cada fila** | `SQRT(sal)` → raíz cuadrada del salario de cada empleado |
| **De agrupamiento** | Devuelven **un solo valor** para un conjunto de filas | `MAX(sal)` → salario máximo de todos los empleados |

---

## Funciones Numéricas

| Función | Descripción | Ejemplo |
|---------|-------------|---------|
| `SQRT(n)` | Raíz cuadrada | `SQRT(81)` → 9 |
| `ABS(n)` | Valor absoluto | `ABS(-11)` → 11 |
| `POWER(base, exp)` | Potencia | `POWER(9, 2)` → 81 |

```sql
SELECT SQRT(sal) FROM emp;
```

---

## Funciones Alfanuméricas

| Función | Descripción | Ejemplo |
|---------|-------------|---------|
| `SUBSTR(cad, ini [, len])` | Subcadena | `SUBSTR('Materia', 2, 4)` → `'ater'` |
| `UPPER(cad)` | Mayúsculas | `UPPER('Materia')` → `'MATERIA'` |
| `LOWER(cad)` | Minúsculas | `LOWER('Materia')` → `'materia'` |
| `LENGTH(cad)` | Longitud | `LENGTH('Materia')` → 7 |

```sql
SELECT UPPER(ename) FROM emp;
```

---

## Funciones de Fecha

| Función | Descripción |
|---------|-------------|
| `CURRENT_DATE` | Fecha actual del sistema |

```sql
SELECT CURRENT_DATE FROM DUAL;
```

> `DUAL` es una tabla especial de Oracle de una sola fila, diseñada para consultas que no requieren datos de tablas reales.

---

## Funciones Específicas de Oracle

| Función | Descripción | Ejemplo |
|---------|-------------|---------|
| `NVL(expr1, expr2)` | Si `expr1` es NULL, devuelve `expr2` | `NVL(sal + comm, sal)` |
| `DECODE(expr, v1, r1, ...)` | Asignación condicional tipo CASE | `DECODE(DEPTNO, '10', 'CONTABILIDAD', '30', 'VENTAS')` |
| `TO_CHAR(fecha, formato)` | Formatea fechas a texto | `TO_CHAR(hiredate, 'yyyy')` |

```sql
-- Si comm es NULL, usa sal como total
SELECT ename, NVL(sal + comm, sal) AS total
FROM emp;
```

---

## CASE WHEN (Expresión Condicional Estándar)

`CASE WHEN` es la expresión condicional **estándar SQL** (ISO/ANSI), más flexible y legible que `DECODE`. Es la forma recomendada en SQL moderno.

**Sintaxis:**
```sql
CASE
    WHEN condición1 THEN resultado1
    WHEN condición2 THEN resultado2
    ...
    [ELSE resultado_por_defecto]
END
```

---

**Comparativa CASE vs DECODE:**

| Característica | `CASE WHEN` | `DECODE` |
|----------------|-------------|----------|
| Estándar SQL | ✅ Sí (ISO/ANSI) | ❌ No (Oracle propietario) |
| Condiciones complejas | ✅ Soporta `>`, `<`, `LIKE`, etc. | ❌ Solo igualdad exacta |
| Legibilidad | ✅ Más clara | ⚠️ Menos intuitiva |
| Portabilidad | ✅ Funciona en todos los SGBD | ❌ Solo Oracle |

---

### Ejemplos de CASE WHEN

**Ejemplo 1: Clasificación de salarios**
```sql
SELECT ename, sal,
    CASE
        WHEN sal >= 3000 THEN 'ALTO'
        WHEN sal >= 1500 THEN 'MEDIO'
        ELSE 'BAJO'
    END AS categoria
FROM emp;
```

| ENAME | SAL | CATEGORIA |
|-------|-----|-----------|
| SMITH | 800 | BAJO |
| ALLEN | 1600 | MEDIO |
| KING | 5000 | ALTO |

> **Ventaja clave**: `CASE WHEN` permite usar cualquier operador de comparación (`>`, `<`, `BETWEEN`, `LIKE`), mientras que `DECODE` solo compara por igualdad exacta.

---

**Ejemplo 2: Equivalente a DECODE con CASE**
```sql
-- Con DECODE (Oracle)
SELECT ename,
    DECODE(deptno, 10, 'CONTABILIDAD',
                    30, 'VENTAS',
                    'OTRO') AS departamento
FROM emp;

-- Con CASE WHEN (estándar SQL)
SELECT ename,
    CASE deptno
        WHEN 10 THEN 'CONTABILIDAD'
        WHEN 30 THEN 'VENTAS'
        ELSE 'OTRO'
    END AS departamento
FROM emp;
```

> La sintaxis `CASE expresión WHEN valor THEN ...` es la forma simple, equivalente a `DECODE`.

---

**Ejemplo 3: CASE con condiciones complejas (imposible con DECODE)**
```sql
-- Bonificación según antigüedad y salario
SELECT ename, hiredate, sal,
    CASE
        WHEN hiredate < '01/01/1981' AND sal < 2000 THEN sal * 0.20
        WHEN hiredate < '01/01/1981' AND sal >= 2000 THEN sal * 0.10
        WHEN hiredate >= '01/01/1981' AND sal < 2000 THEN sal * 0.15
        ELSE sal * 0.05
    END AS bonificacion
FROM emp;
```

> Este tipo de lógica condicional compleja (múltiples condiciones con `AND`/`OR`) es **imposible** de expresar con `DECODE`.

---

**Ejemplo 4: CASE en cláusulas ORDER BY**
```sql
-- Ordenar por categoría personalizada
SELECT ename, job
FROM emp
ORDER BY
    CASE job
        WHEN 'PRESIDENT' THEN 1
        WHEN 'MANAGER' THEN 2
        WHEN 'ANALYST' THEN 3
        WHEN 'SALESMAN' THEN 4
        ELSE 5
    END;
```

> `CASE WHEN` puede usarse en `SELECT`, `WHERE`, `ORDER BY`, `HAVING` e incluso en funciones de agrupamiento.

---

**Ejemplo 5: CASE con funciones de agrupamiento**
```sql
-- Contar empleados por rango de salario
SELECT
    CASE
        WHEN sal < 1500 THEN '< 1500'
        WHEN sal BETWEEN 1500 AND 3000 THEN '1500-3000'
        ELSE '> 3000'
    END AS rango_salarial,
    COUNT(*) AS num_empleados,
    AVG(sal) AS salario_medio
FROM emp
GROUP BY
    CASE
        WHEN sal < 1500 THEN '< 1500'
        WHEN sal BETWEEN 1500 AND 3000 THEN '1500-3000'
        ELSE '> 3000'
    END;
```

> **Regla importante**: Si usas `CASE` en `SELECT` con `GROUP BY`, debes repetir exactamente la misma expresión `CASE` en `GROUP BY` (no puedes usar el alias).

---

### ¿Cuándo usar CASE WHEN vs DECODE?

| Situación | Recomendación |
|-----------|---------------|
| Comparaciones simples por igualdad | `DECODE` (si es código legacy Oracle) o `CASE` |
| Comparaciones con `<`, `>`, `BETWEEN`, `LIKE` | `CASE WHEN` obligatoriamente |
| Múltiples condiciones combinadas (`AND`/`OR`) | `CASE WHEN` obligatoriamente |
| Código portable entre SGBD | `CASE WHEN` obligatoriamente |
| Mantenimiento de código legacy Oracle | `DECODE` (para consistencia) |
| Nuevos desarrollos | `CASE WHEN` (mejor práctica) |

# 5. Funciones de Agrupamiento

## Definición

Las funciones de agrupamiento (también llamadas agregadas o colectivas) obtienen un **único valor** que resume un **conjunto de filas**.

**Sintaxis:**
```sql
<funcion>([ALL | DISTINCT] <expresion>)
```

---

| Función | Descripción |
|---------|-------------|
| `AVG(expr)` | Media aritmética |
| `COUNT(expr)` | Número de valores no NULL |
| `COUNT(*)` | Número total de filas |
| `MAX(expr)` | Valor máximo |
| `MIN(expr)` | Valor mínimo |
| `SUM(expr)` | Suma de valores |
| `VAR(expr)` | Varianza |

> `DISTINCT` elimina valores repetidos antes de calcular. `ALL` es el comportamiento por defecto.

---

## La Función COUNT en Detalle

| Formato | Qué cuenta |
|---------|------------|
| `COUNT(*)` | Todas las filas (incluye NULLs) |
| `COUNT(columna)` | Filas donde la columna no es NULL |
| `COUNT(ALL columna)` | Igual que COUNT(columna) |
| `COUNT(DISTINCT columna)` | Valores distintos y no NULL |

**Ejemplo práctico:**
```sql
-- Tabla t1 con columna C: NULL, 1, 2, 2
SELECT COUNT(*), COUNT(c), COUNT(ALL c), COUNT(DISTINCT c)
FROM t1;
-- Resultado: 4, 3, 3, 2
```

---

## Ejemplos de Funciones de Agrupamiento

```sql
-- Media de salarios del departamento 30
SELECT AVG(sal) FROM emp WHERE deptno = 30;
```

```sql
-- Media de salarios y nombre más alto alfabéticamente
-- de empleados con salario > 1500
SELECT AVG(sal), MAX(ename)
FROM emp WHERE sal > 1500;
```

```sql
-- Media de todos los salarios
SELECT AVG(sal) FROM emp;
```

> **Regla importante**: Si incluyes una función de agrupamiento en `SELECT`, todas las demás expresiones en `SELECT` deben tener un **valor único para el conjunto**.

# 6. Cláusula GROUP BY

## Definición

`GROUP BY` permite formar **grupos de filas** que tienen los mismos valores en una o más columnas (llamadas **columnas de agrupamiento**).

**Funcionamiento:**
1. El SGBD divide las filas en grupos según igualdad de valores en las columnas de agrupamiento
2. Las funciones de agrupamiento se aplican a cada grupo por separado
3. El resultado contiene **una fila por grupo**

---

## Regla Fundamental de GROUP BY

> **REGLA DE ORO**: Si una consulta incluye `GROUP BY`, en la cláusula `SELECT` solo pueden aparecer:
> 1. **Constantes**
> 2. **Columnas de agrupamiento** (las que están en GROUP BY)
> 3. **Funciones de agrupamiento**
> 4. **Combinaciones** de los anteriores

**Ejemplo CORRECTO:**
```sql
SELECT MAX(sal), deptno        -- ✅ MAX() es función de agrupamiento
FROM emp                        -- ✅ deptno es columna de agrupamiento
GROUP BY deptno;
```

**Ejemplo INCORRECTO:**
```sql
SELECT MAX(sal), deptno, empno  -- ❌ empno NO es columna de agrupamiento
FROM emp                        --    ni función de agrupamiento
GROUP BY deptno;                --    → ERROR: empno tiene múltiples valores por grupo
```

---

## Ejemplos de GROUP BY

```sql
-- Máximo, mínimo, suma y número de empleados por departamento
SELECT MAX(sal), MIN(sal), SUM(sal), COUNT(empno), deptno
FROM emp
GROUP BY deptno;
```

| MAX(SAL) | MIN(SAL) | SUM(SAL) | COUNT(EMPNO) | DEPTNO |
|----------|----------|----------|--------------|--------|
| 5000 | 1300 | 8750 | 3 | 10 |
| 3000 | 800 | 10875 | 5 | 20 |
| 2850 | 950 | 9400 | 6 | 30 |

---

```sql
-- Lo mismo, pero solo para empleados con salario > 1500
SELECT MAX(sal), MIN(sal), SUM(sal), COUNT(empno), deptno
FROM emp
WHERE sal > 1500
GROUP BY deptno;
```

> `WHERE` se aplica **antes** de agrupar; filtra filas, no grupos.

---

## GROUP BY con Múltiples Columnas

```sql
-- Empleos diferentes por departamento, con conteo
SELECT job, deptno, COUNT(*)
FROM emp
GROUP BY deptno, job
ORDER BY job, deptno;
```

<div class=two-cols>

<div>

| JOB | DEPTNO | COUNT(*) |
|-----|--------|----------|
| ANALYST | 20 | 2 |
| CLERK | 10 | 1 |
| CLERK | 20 | 2 |
| MANAGER | 10 | 1 |

</div>
<div>

| JOB | DEPTNO | COUNT(*) |
|-----|--------|----------|
| MANAGER | 20 | 1 |
| MANAGER | 30 | 1 |
| PRESIDENT | 10 | 1 |
| SALESMAN | 30 | 4 |

</div>

> El orden de columnas en `GROUP BY` no afecta al resultado, solo a la presentación. Se crea un grupo por cada combinación única de valores.

# 7. Cláusula HAVING

## Definición

`HAVING` establece **condiciones sobre los grupos** formados por `GROUP BY`, a diferencia de `WHERE` que filtra filas individuales.

**Características:**
- Solo puede usarse cuando hay una cláusula `GROUP BY`
- Las condiciones deben ser aplicables a cada grupo (no a filas individuales)
- Puede incluir funciones de agrupamiento

---

## WHERE vs HAVING - Comparativa

| Característica | WHERE | HAVING |
|----------------|-------|--------|
| **Momento de ejecución** | Antes de agrupar | Después de agrupar |
| **Aplica a** | Filas individuales | Grupos de filas |
| **Puede usar funciones de agrupamiento** | ❌ No | ✅ Sí |
| **Puede usar columnas no agrupadas** | ✅ Sí | ❌ No (solo constantes, columnas de agrupamiento o funciones de agrupamiento) |

---

## Ejemplos de HAVING

```sql
-- Departamentos con más de 3 empleados
SELECT MAX(sal), deptno
FROM emp
GROUP BY deptno
HAVING COUNT(*) > 3;
```

```sql
-- Departamentos donde el salario mínimo es mayor a 1600
SELECT MAX(sal), MIN(sal), SUM(sal), COUNT(empno), deptno
FROM emp
WHERE sal > 1500
GROUP BY deptno
HAVING MIN(sal) > 1600;
```

| MAX(SAL) | MIN(SAL) | SUM(SAL) | COUNT(EMPNO) | DEPTNO |
|----------|----------|----------|--------------|--------|
| 5000 | 2450 | 7450 | 2 | 10 |
| 3000 | 2975 | 8975 | 3 | 20 |

---

## Ejemplo Combinado: WHERE + HAVING

```sql
SELECT MAX(sal), deptno
FROM emp
WHERE sal > 1500          -- Primero: filtra filas individuales
GROUP BY deptno
HAVING COUNT(*) > 3;      -- Luego: filtra grupos resultantes
```

**Flujo de ejecución:**
1. `FROM emp` → Toma la tabla
2. `WHERE sal > 1500` → Elimina filas con salario <= 1500
3. `GROUP BY deptno` → Agrupa las filas restantes por departamento
4. `HAVING COUNT(*) > 3` → Elimina grupos con 3 o menos empleados
5. `SELECT MAX(sal), deptno` → Calcula máximo por grupo
6. `ORDER BY` (si existiera) → Ordena resultado

# 8. Subconsultas

## Definición

Una **subconsulta** (o consulta anidada) es una sentencia `SELECT` incluida dentro de otra consulta (llamada **consulta principal** o **consulta externa**).

**Características:**
- El resultado de la subconsulta se usa en un predicado de la consulta principal
- No aparece directamente en el resultado final
- Puede haber subconsultas anidadas a varios niveles

---

## Orden de Ejecución en Subconsultas

> **Regla general**: La subconsulta se **ejecuta primero**, se sustituye por su resultado, y luego se evalúa la consulta principal.

**Proceso:**
1. Se ejecuta la subconsulta
2. Se obtiene su resultado (escalar, una fila o varias filas)
3. Se sustituye el resultado en el predicado de la consulta principal
4. Se ejecuta la consulta principal con el valor sustituido

**Excepción**: Las **subconsultas correlacionadas** se ejecutan una vez por cada fila de la consulta externa.

---

## Tipos de Subconsultas según Resultado

| Tipo | Descripción | Operadores válidos |
|------|-------------|-------------------|
| **Escalar** | Una fila, una columna | `=`, `<`, `>`, `<=`, `>=`, `!=` |
| **Una fila, varias columnas** | Una fila, múltiples columnas | `=`, `IN` (con tuplas) |
| **Varias filas, una columna** | Múltiples filas, una columna | `IN`, `ANY`, `ALL`, `EXISTS` |

---

## Subconsulta Escalar

```sql
-- Empleados cuyo salario es mayor que el del empleado 7900
SELECT empno, ename, sal
FROM emp
WHERE sal > (SELECT sal FROM emp WHERE empno = 7900);
```

**Ejecución:**
1. Subconsulta: `SELECT sal FROM emp WHERE empno = 7900` → devuelve 950
2. Consulta principal: `SELECT ... WHERE sal > 950`

> Una subconsulta escalar **debe devolver exactamente una fila y una columna**. Si devuelve más de una fila, se produce un error.

---

## Subconsultas con Operadores Cuantificados

Cuando la subconsulta devuelve **varias filas**, se usan operadores cuantificados:

| Operador | Significado |
|----------|-------------|
| `ANY` / `SOME` | Verdadero si la comparación es cierta para **algún** valor de la subconsulta |
| `ALL` | Verdadero si la comparación es cierta para **todos** los valores de la subconsulta |

---

```sql
-- Empleados con salario igual a alguno del dept 30
SELECT * FROM emp
WHERE sal = ANY (SELECT sal FROM emp WHERE deptno = 30);

-- Equivalente a:
SELECT * FROM emp
WHERE sal IN (SELECT sal FROM emp WHERE deptno = 30);
```

```sql
-- Empleados con salario mayor que TODOS los del dept 30
SELECT * FROM emp
WHERE sal > ALL (SELECT sal FROM emp WHERE deptno = 30);
```

---

## Subconsultas de Fila (Row Subqueries)

Comparan múltiples columnas simultáneamente:

```sql
SELECT *
FROM articulo
WHERE (cd_articulo, precio_min)
      IN (SELECT cd_articulo, precio FROM ventas);
```

> En Oracle 9.2, operadores válidos: `IN`, `=SOME`, `=ALL`

---

## Tablas Derivadas (Subconsultas en FROM)

Una consulta puede usarse como **tabla derivada** en la cláusula `FROM`:

```sql
-- Media de los salarios medios de los departamentos 10 y 20
SELECT AVG(PREMED) AS MEDIA
FROM (SELECT AVG(sal) AS PREMED
      FROM emp
      WHERE deptno IN (10, 20)
      GROUP BY deptno) AS tabla_derivada;
```

> También se puede escribir con funciones anidadas: `SELECT AVG(AVG(sal)) FROM emp ... GROUP BY deptno`

---

## Predicado EXISTS

`EXISTS` comprueba si una subconsulta devuelve **alguna fila** (al menos una).

```sql
-- Si existen empleados en el departamento 30, muestra todos los empleados
SELECT *
FROM emp
WHERE EXISTS (SELECT * FROM emp WHERE deptno = 30);
```

**Características:**
- Solo comprueba la **existencia** de filas, no sus valores
- Es eficiente: puede detenerse al encontrar la primera coincidencia
- La subconsulta puede tener varias expresiones en SELECT

---

## Subconsultas Correlacionadas

Una subconsulta es **correlacionada** cuando hace referencia a columnas de la tabla de la consulta principal.

**Características:**
- Se ejecuta **una vez por cada fila** de la consulta externa
- Requiere **alias de tabla** para distinguir tablas interna y externa
- El valor de la subconsulta depende de la fila actual de la consulta externa

<br>

```sql
-- Empleados que tienen subordinados
SELECT a.*
FROM emp a
WHERE EXISTS (SELECT * FROM emp b WHERE b.mgr = a.empno);
```
---

```sql
-- Empleados cuyo salario supera la media de SU departamento
SELECT *
FROM emp a
WHERE sal > (SELECT AVG(sal)
               FROM emp
               WHERE deptno = a.deptno);
```

**Ejecución:**
1. Toma la primera fila de `emp a` (ej: SMITH, deptno=20, sal=800)
2. Ejecuta subconsulta: `SELECT AVG(sal) FROM emp WHERE deptno = 20` → 2175
3. Compara: 800 > 2175? → FALSE → No incluye a SMITH
4. Toma la siguiente fila y repite...

# 9. JOINs

## Definición Conceptual

Un **JOIN** (o combinación) es una operación que permite obtener datos de **más de una tabla** simultáneamente, vinculando filas mediante una condición de relación.

**Concepto clave:**
- Permite "unir" tablas relacionadas a través de columnas comunes
- El resultado es una tabla virtual con columnas de ambas tablas
- La condición de vinculación se denomina **condición de join**

---

**Tipos principales:**
| Tipo | Descripción |
|------|-------------|
| `INNER JOIN` | Solo filas que cumplen la condición en AMBAS tablas |
| `LEFT OUTER JOIN` | Todas las filas de la izquierda + coincidencias de la derecha |
| `RIGHT OUTER JOIN` | Todas las filas de la derecha + coincidencias de la izquierda |
| `FULL OUTER JOIN` | Todas las filas de ambas tablas |

---

## INNER JOIN

Combina filas de dos tablas donde se cumple la condición de vinculación.

**Sintaxis estándar SQL:**
```sql
SELECT *
FROM emp INNER JOIN dept
  ON emp.deptno = dept.deptno
WHERE ename LIKE 'A%';
```

**Sintaxis clásica Oracle (coma):**
```sql
SELECT *
FROM emp, dept
WHERE emp.deptno = dept.deptno
  AND ename LIKE 'A%';
```

> Solo aparecen las filas que **cumplen la condición en ambas tablas**. Las filas sin correspondencia se descartan.

---

## OUTER JOIN

Permite que aparezcan **todas las filas** de una tabla, rellenando con `NULL` las columnas de la otra tabla cuando no hay correspondencia.

| Tipo | Resultado |
|------|-----------|
| `LEFT OUTER JOIN` | Todas las filas de la tabla **izquierda** |
| `RIGHT OUTER JOIN` | Todas las filas de la tabla **derecha** |
| `FULL OUTER JOIN` | Todas las filas de **ambas** tablas |

```sql
-- Todos los departamentos, incluso sin empleados
SELECT *
FROM dept LEFT OUTER JOIN emp
  ON dept.deptno = emp.deptno;
```

> El departamento 40 (OPERATIONS) aparecerá con valores NULL en las columnas de empleado.

---

## Múltiples JOINs

El orden de ejecución es de **izquierda a derecha** (asociativo a la izquierda):

```sql
SELECT ...
FROM t1
JOIN t2 ON <condición_join1>
JOIN t3 ON <condición_join2>
WHERE <predicado>;
```

**Equivalente con paréntesis:**
```sql
SELECT ...
FROM (t1 JOIN t2 ON <condición_join1>)
JOIN t3 ON <condición_join2>
WHERE <predicado>;
```
--- 

Para modificar el orden de evaluación, usa paréntesis explícitos:
```sql
SELECT ...
FROM t1
JOIN (t2 JOIN t3 ON <condición_join2>)
  ON <condición_join1>
WHERE <predicado>;
```

# 11. Otras Sentencias DML

## INSERT

Añade filas a una tabla. No garantiza posición física (no siempre al final).

```sql
-- Insertar todos los valores en orden de columnas
INSERT INTO emp
VALUES (7777, 'ANA', 'ANALISTA', 7369, '15/01/05', 2500, NULL, 10);

-- Insertar solo columnas específicas (las demás quedan NULL o con valor por defecto)
INSERT INTO emp (empno, ename, sal)
VALUES (8888, 'Juan', 2000);

-- Insertar resultado de una consulta
INSERT INTO emp_new
SELECT empno, ename, sal, comm
FROM emp_old
WHERE deptno = 10 AND sal > 1500;
```

---

## UPDATE

Modifica datos existentes en filas.

```sql
-- Actualizar TODAS las filas (¡cuidado!)
UPDATE emp SET sal = sal + 100;

-- Actualizar filas que cumplen condición
UPDATE emp
SET comm = 110, deptno = 10
WHERE comm IS NULL;

-- Actualizar usando subconsulta
UPDATE emp
SET sal = 2000
WHERE sal = (SELECT MIN(sal) FROM emp a WHERE a.deptno = emp.deptno);
```

> **¡Advertencia!** Sin `WHERE`, se actualizan **todas** las filas de la tabla.

---

## DELETE

Elimina filas de una tabla.

```sql
-- Borrar filas que cumplen condición
DELETE FROM emp WHERE sal > 2000;

-- ¡BORRAR TODAS LAS FILAS! (¡extremo cuidado!)
DELETE FROM emp;
```

> **¡Advertencia!** Sin `WHERE`, se eliminan **todas** las filas. No se puede deshacer sin ROLLBACK si no se ha hecho COMMIT.

# Resumen: Orden de Ejecución de SELECT

```
┌─────────────────────────────────────────┐
│  1. FROM      → Determina tablas        │
│  2. WHERE     → Filtra filas            │
│  3. GROUP BY  → Agrupa filas            │
│  4. HAVING    → Filtra grupos           │
│  5. SELECT    → Selecciona columnas     │
│  6. ORDER BY  → Ordena resultado        │
└─────────────────────────────────────────┘
```

**Reglas clave:**
- `WHERE` filtra **antes** de agrupar (filas)
- `HAVING` filtra **después** de agrupar (grupos)
- En `SELECT` con `GROUP BY`: solo constantes, columnas de agrupamiento o funciones de agrupamiento
- Subconsultas escalares se ejecutan primero (excepto correlacionadas)

---

# Resumen: JOINs

| Tipo | Qué devuelve | Uso típico |
|------|-------------|------------|
| `INNER JOIN` | Solo coincidencias | Datos relacionados que existen en ambas tablas |
| `LEFT JOIN` | Izquierda completa + coincidencias | Mostrar todos los registros de la tabla principal |
| `RIGHT JOIN` | Derecha completa + coincidencias | Mostrar todos los registros de la tabla secundaria |
| `FULL JOIN` | Ambas tablas completas | Combinar datos con posibles ausencias en ambos lados |

---

# Consejos Prácticos

- ✅ Usa `WHERE` para filtrar antes de agrupar (más eficiente)
- ✅ Usa `HAVING` solo para condiciones sobre funciones de agrupamiento
- ✅ Recuerda: cualquier operación con NULL produce NULL
- ✅ Usa `IS NULL` / `IS NOT NULL`, nunca `= NULL` o `!= NULL`
- ✅ En subconsultas correlacionadas, usa alias de tabla obligatoriamente
- ✅ Verifica resultados con `SELECT` antes de hacer `UPDATE` o `DELETE`
- ✅ Usa `EXISTS` en lugar de `IN` cuando la subconsulta pueda devolver muchas filas
- ✅ Un JOIN conceptualmente crea el producto cartesiano y luego filtra
- ✅ Prefiere `CASE WHEN` sobre `DECODE` en nuevos desarrollos (estándar SQL y más flexible)
