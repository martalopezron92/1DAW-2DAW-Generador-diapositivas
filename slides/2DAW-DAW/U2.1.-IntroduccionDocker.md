---

# ¿Qué es Docker?

Docker es una plataforma de código abierto que permite **automatizar el despliegue de aplicaciones dentro de contenedores**.

Un contenedor incluye todo lo necesario para ejecutar una aplicación:

- código
- runtime
- herramientas del sistema
- bibliotecas
- configuración

---

# Docker y contenedores

Docker proporciona una capa adicional de abstracción y automatización de virtualización a nivel de sistema operativo.

### Idea clave

- Los contenedores Docker **comparten el kernel** del sistema operativo host.
- Por eso son **más ligeros y eficientes** que las máquinas virtuales tradicionales.

---

# Contenedores vs máquinas virtuales

| Aspecto | Contenedores Docker | Máquinas virtuales tradicionales |
|---|---|---|
| Base de funcionamiento | Comparten el kernel del host | No comparten el kernel del host |
| Peso | Más ligeros | Más sobrecarga |
| Arranque | En segundos | Más lento |
| Eficiencia | Alta | Menor |

---

# Ventajas de usar Docker

- **Portabilidad**: funcionan igual en cualquier entorno
- **Eficiencia**: menos sobrecarga
- **Aislamiento**: cada contenedor se ejecuta de forma aislada
- **Rapidez**: se inician en segundos
- **Consistencia**: mismo entorno en desarrollo, pruebas y producción

---

# Instalación de Docker Engine

Docker Engine está disponible en:

- varias distribuciones de **Linux**
- **macOS**
- **Windows 10** mediante Docker Desktop

En este tema, la instalación se realiza en **Debian/Ubuntu**.

---

# Instalación en Windows con Docker Desktop

En Windows, Docker se utiliza normalmente mediante **Docker Desktop**.

#### Pasos generales

1. Descargar **Docker Desktop**.
2. Ejecutar el instalador.
3. Completar el asistente de instalación.
4. Reiniciar el equipo si el sistema lo solicita.
5. Abrir Docker Desktop y esperar a que quede iniciado.

#### Comprobación básica

```bash
docker --version
docker run hello-world
```

---

# Instalación en Linux · Paso 1

#### 1) Actualizar repositorios e instalar prerequisitos

```bash
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg
```

#### 2) Añadir la clave GPG oficial de Docker

```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
```

---

# Instalación en Linux · Paso 2

#### 3) Añadir el repositorio oficial

```bash
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
$(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

#### 4) Instalar Docker Engine

```bash
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

---

# Instalación en Linux · Paso 3

#### 5) Usar Docker sin privilegios

```bash
sudo usermod -aG docker $USER
su - $USER
```

#### 6) Verificar la instalación

```bash
docker --version
```

Salida esperada:

```text
Docker version 24.0.7, build afdd53b
```

## Tu primer contenedor: `hello-world`

```bash
docker run hello-world
```

Salida típica:

```bash
Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
This message shows that your installation appears to be working correctly.
```

---

# ¿Qué ha sucedido? Paso a paso

1. Docker busca la imagen `hello-world` en tu ordenador.
2. Si no la encuentra, la descarga de Docker Hub.
3. Descomprime sus capas.
4. Crea un nuevo contenedor.
5. Ejecuta el programa predefinido.
6. Muestra la salida en el terminal.
7. El contenedor se detiene al terminar.

> **Consejo:** la primera ejecución tarda más porque hay que descargar la imagen.

# Opciones útiles de `docker ps`

```bash
docker ps
docker ps -a
docker ps -q
```

- `docker ps` → solo contenedores en ejecución
- `docker ps -a` → todos los contenedores
- `docker ps -q` → solo los IDs

> **Nota importante:** un contenedor ejecuta un proceso y, cuando ese proceso termina, el contenedor se para.

# Contenedores interactivos

```bash
docker run -it --name contenedor1 ubuntu bash
```

#### Significado de las opciones

- `-i` → mantiene abierta la entrada estándar
- `-t` → asigna un pseudo-terminal
- `--name contenedor1` → pone un nombre al contenedor
- `ubuntu` → imagen base
- `bash` → intérprete de comandos que se ejecuta

---

# `attach` y `exec`: idea práctica

#### `docker attach`

- te conecta a la terminal de un contenedor ya en ejecución

Para salir **sin detenerlo**:

- `Ctrl+P`
- luego `Ctrl+Q`

#### `docker exec`

- ejecuta un comando en un contenedor que **ya está ejecutándose**
- muestra la salida en tu terminal

# Contenedores en segundo plano

Muchos servicios deben ejecutarse continuamente. Para eso usamos `-d`:

```bash
docker run -d --name servidor_web nginx
```

#### Significado

- `-d` → **detached mode**
- el contenedor se ejecuta en segundo plano
- el terminal devuelve el ID del contenedor

#### Comprobación

```bash
docker ps
```

Si todo va bien, `STATUS` mostrará algo como `Up 4 seconds`.

---

# Ver logs y detener un servicio

#### Logs

```bash
docker logs servidor_web
docker logs -f servidor_web
docker logs --tail 50 servidor_web
docker logs -t servidor_web
```

#### Detener correctamente o forzar cierre inmediato

<div class="two-cols">
<div>

```bash
docker stop servidor_web
```

</div>
<div>

```bash
docker kill servidor_web
```

</div>
</div>

- envía `SIGTERM`
- espera 10 segundos
- si no termina, fuerza el cierre

> **Buena práctica:** usar `docker stop` antes que `docker kill`.

# Servidor web con mapeo de puertos

```bash
docker run -d --name mi_nginx -p 8080:80 nginx
```

#### Interpretación de `-p 8080:80`

- `8080` → puerto en tu ordenador
- `80` → puerto dentro del contenedor

#### Flujo

`localhost:8080` → `Docker` → `contenedor:80` → `Nginx`

---

# Cómo comprobar el mapeo

<div class="two-cols">
<div>

Comando 

```bash
docker ps
```
</div>
<div>

Ejemplo de salida:

```text
0.0.0.0:8080->80/tcp
```
</div>
</div>

#### Significado

- `0.0.0.0` → escucha en todas las interfaces
- `8080->80` → puerto host → puerto contenedor
- `tcp` → protocolo TCP

#### Múltiples puertos

```bash
docker run -d --name mi_nginx_ssl -p 8080:80 -p 8443:443 nginx
```

> **Advertencia:** si el puerto `8080` ya está ocupado, usa otro, por ejemplo `9090:80`.

# Configuración con variables de entorno

Las variables de entorno son el método estándar para configurar contenedores.

<div class="two-cols">
<div>

#### Ejemplo con MySQL

```bash
docker run -d --name mi_mysql \
  -e MYSQL_ROOT_PASSWORD=secreto \
  -e MYSQL_DATABASE=midb \
  -e MYSQL_USER=usuario \
  -e MYSQL_PASSWORD=pass123 \
  mysql:5.7
```
</div>
<div>

#### Qué permiten

- definir contraseña de `root`
- crear una base de datos
- crear un usuario
- asignar su contraseña

</div>
</div>

---

# Cómo comprobar la configuración

#### Ver variables del contenedor

```bash
docker exec mi_mysql env
```

#### Conectarte a la base de datos

```bash
docker exec -it mi_mysql mysql -u usuario -ppass123 midb
```

Dentro de MySQL:

```bash
SHOW DATABASES;
exit
```

> **Seguridad:** no pongas contraseñas reales en la línea de comandos. Usa archivos `.env` o secrets de Docker.

---

# Misma imagen, distinta configuración

```bash
# Desarrollo
docker run -d --name app_dev \
  -e APP_ENV=development \
  -e DEBUG=true \
  -e LOG_LEVEL=debug \
  mi_aplicacion
```

```bash
# Producción
docker run -d --name app_prod \
  -e APP_ENV=production \
  -e DEBUG=false \
  -e LOG_LEVEL=warning \
  mi_aplicacion
```

#### Idea clave

La **misma imagen** puede comportarse de forma distinta según las variables de entorno.

# Más comandos de gestión

#### Eliminar contenedores

```bash
docker rm contenedor1
docker rm -f mi_nginx
docker container prune
```

#### Contenedores temporales

```bash
docker run --rm ubuntu echo "Hola"
```

#### Inspección y recursos

```bash
docker inspect contenedor1
docker stats contenedor1
```

**docker rm -f fuerza la eliminación y debe usarse solo cuando sea necesario.**

---

# Ciclo de vida de un contenedor

1. **Crear** → `docker create`
2. **Iniciar** → `docker start`
3. **Ejecutar** → `docker run`
4. **Pausar** → `docker pause`
5. **Reanudar** → `docker unpause`
6. **Detener** → `docker stop`
7. **Matar** → `docker kill`
8. **Eliminar** → `docker rm`

#### Idea visual

`crear → iniciar/ejecutar → pausar/reanudar → detener → eliminar`
